# UI はフローティングバー1つに集約する

pasha-go は伝統的なメインウィンドウを持たず、macOS の `Cmd+Shift+5` のような常時表示のフローティングバー1本に**設定（Repeat Count・Step Interval・出力先）と操作（範囲選択・クリック位置選択・開始・停止）を全て集約**する。

Capture Session 実行中もバーは表示され続け、「停止」を常に1クリックで届くようにする。デスクトップ自動化ツールという性質上、ユーザーは常にターゲットアプリを見ており、メインウィンドウに戻る動線はむしろ邪魔。バーが小さく済む代わりに、入力ウィジェットのレイアウト設計には工夫が要る。

## Considered Options

- メインウィンドウ + 実行用フローティングバーの2ウィンドウ構成：設定とアクションを綺麗に分離できるが、2ウィンドウ管理が必要
- フローティングバー + 設定ドロワー（中庸）
- **フローティングバー1つに集約（採用）**：Cmd+Shift+5 のメタファに最も忠実
